How CinePick works

Content-based recommendations using text features from each film—not collaborative filtering.

Pipeline

  1. Dataset — Each row is a movie with metadata: title, overview, genres, keywords, cast, crew, and a combined text field called soup.
  2. Vectorization — English stop words are removed; the rest is turned into a sparse bag-of-words matrix with CountVectorizer (scikit-learn).
  3. Similarity — Cosine similarity compares your movie’s vector to every other movie. The top matches (excluding itself) become recommendations.
  4. Logging — When you search or explore, the title is appended to movieR.csv for a simple interaction history.

Fuzzy angle

The “fuzzy” part of the project name refers to soft, overlapping similarity in content space: many weak word overlaps add up to a strong cosine score, rather than a single exact keyword match. It is not the same as fuzzy string matching on the title alone.

Images

Posters use placeholder photography (Picsum) keyed by TMDB-style id so each film gets a stable image. The bundled CSV does not include poster_path; wiring the real TMDB image CDN would need that field or an API key.

Tech stack

  • Flask — web UI and routing
  • pandas — CSV loading
  • scikit-learn — CountVectorizer, cosine_similarity

Screens

  • Dashboard — overview and shortcuts
  • Search — pick a title and submit
  • Explore — random history-based run
  • Results — opens after a successful search
